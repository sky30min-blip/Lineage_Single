# 서버 폴더 용량 정리 – 필수 vs 불필요 파일

`2.싱글리니지 팩` 기준으로, **구동에 꼭 필요한 것**과 **지워도 되거나 정리하면 좋은 것**을 구분했습니다.

---

## 1. 게임 서버 구동에 **꼭 필요한 것**

| 폴더/파일 | 용도 | 대략 용량 |
|-----------|------|-----------|
| **server.jar** | 서버 메인 실행 파일 (빌드 결과물) | ~수 MB |
| **lib/** | JDBC, Netty, Log4j 등 JAR 라이브러리 | ~8 MB |
| **lineage.conf** | 서버 설정 | - |
| **lineage_balance.conf** | 밸런스 설정 | - |
| **log.conf, log4j.conf** | 로그 설정 | - |
| **mysql.conf** | DB 접속 설정 | - |
| **socket.conf** | 소켓 설정 | - |
| **npc_promotion.conf** | NPC 프로모션 설정 | - |
| **maps/** | 맵 데이터 (게임 맵) | ~46 MB |
| **db/** | DB 스크립트 (초기 설정/업데이트용) | ~9 MB |
| **Font/** | 폰트 리소스 | ~5 MB |
| **resource/** | 리소스 파일 | ~3 MB |
| **kubera_word/** | 금지어 등 | - |
| **images/** (서버 쪽) | 서버용 이미지 리소스 | - |

→ 위 항목은 **삭제하면 서버가 안 뜨거나 설정/맵/DB 적용이 깨질 수 있으니 유지**하는 게 좋습니다.

---

## 2. **빌드할 때만** 쓰는 것 (구동에는 불필요)

| 폴더/파일 | 용도 | 대략 용량 | 비고 |
|-----------|------|-----------|------|
| **src/** | Java 소스 코드 | ~6.5 MB | 수정 후 다시 빌드할 때만 필요. 구동은 server.jar만 사용 |
| **build/** | Ant 빌드 시 생성되는 .class 모음 | ~5.5 MB | `build.xml`로 빌드하면 생기고, `clean2`로 지움. **삭제 가능** (다음 빌드 시 다시 생김) |
| **apache-ant-1.10.14/** | Ant 빌드 도구 | **~44 MB** | `ant -f build.xml` 로 빌드할 때만 필요. **compile.bat만 쓰면 삭제해도 됨** (용량 많이 줄어듦) |
| **build.xml** | Ant 빌드 스크립트 | - | Ant로 빌드할 때만 필요 |
| **compile.bat, build.ps1** | 컴파일/빌드 스크립트 | - | 빌드할 때만 사용 |

→ **실행만** 할 거면 `server.jar` + `lib/` + 설정/맵/리소스만 있으면 되고,  
→ **코드 수정 없이** 빌드도 안 할 거면 `build/`, `apache-ant-1.10.14/` 삭제해도 됩니다.

---

## 3. **중복·불필요**로 정리해도 되는 것

| 폴더/파일 | 설명 | 대략 용량 | 조치 |
|-----------|------|-----------|------|
| **bin/** | IDE(Eclipse 등)가 만든 .class 출력 폴더 | **~6.4 MB** | 서버는 **server.jar**만 사용. **bin은 구동에 안 씀 → 삭제 가능** (용량 확보) |
| **log/** | 로그 파일들 | **~9.2 MB** | 예전 로그. **오래된 로그 삭제 또는 보관 후 정리**하면 용량 줄어듦 |
| **sql/** 안의 대용량 파일 | list.spr, spr_frame.sql 등 | **~10 MB** | DB/툴용이면 보관용으로만 두고, 서버 구동에 필수인지 확인 후 불필요하면 이동/삭제 검토 |

---

## 4. **선택 정리** (백업/유틸)

| 항목 | 설명 | 조치 |
|------|------|------|
| **mysql.conf.bak, socket.conf.bak** | 설정 백업 | 필요 없으면 삭제 |
| **remove_unused_imports.py** | 코드 정리용 스크립트 | 서버 구동과 무관. 보관할지 삭제할지 선택 |
| **sources.txt, sources_build.txt, sources_cp949.txt** | 소스 목록 등 | 빌드/구동에 불필요. 필요 없으면 삭제 |
| **notice.txt** | 공지용 | 사용하는 공지면 유지 |

---

## 5. 요약 – “지우면 용량이 많이 줄어드는 것” 우선순위

1. **apache-ant-1.10.14/** 삭제  
   - Ant로 빌드 안 하면 **약 44 MB** 절약.
2. **bin/** 삭제  
   - **약 6 MB** 절약. (실행은 server.jar만 사용)
3. **build/** 삭제  
   - **약 5.5 MB** 절약. (다음에 Ant 빌드하면 다시 생김)
4. **log/** 안 오래된 로그 정리  
   - **최대 약 9 MB** 정도 절감 가능.

정리하면, **구동에 꼭 필요한 건 server.jar, lib/, 설정 파일들, maps/, db/, Font/, resource/, kubera_word 등**이고,  
**안 쓸 거면 apache-ant, bin, build, 오래된 log**부터 지우면 서버 폴더 용량이 많이 줄어듭니다.

---

## 6. 주의사항

- **server.jar**를 지우면 서버가 실행되지 않습니다.  
  - jar를 지운 뒤에는 반드시 `compile.bat` 또는 Ant로 다시 빌드해 server.jar를 만들어야 합니다.
- **bin/** 을 지워도 **실행에는 영향 없음** (현재 배치/설정은 server.jar 기준).
- **build/** 는 Ant로 다시 빌드하면 자동으로 다시 생성됩니다.
- **log/** 는 문제 추적용으로 최근 것만 남기고 오래된 것만 정리하는 것을 권장합니다.

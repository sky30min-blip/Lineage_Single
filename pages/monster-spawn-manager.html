🎮 몬스터 스폰 관리

사냥터/던전별 몬스터 배치를 쉽게 관리하세요

1️⃣ 사냥터/던전 선택
2️⃣ 몬스터 선택 (검색창에 입력 시 해당 단어로 필터)
3️⃣ 스폰 설정

💡 범위 설정 시 개체수만큼 자동으로 좌표 분산 배치됩니다

4️⃣ SQL 쿼리 미리보기
-- SQL 쿼리가 여기에 표시됩니다
위에서 설정을 완료하면 실행될 쿼리가 표시됩니다.
5️⃣ 현재 맵의 스폰 목록
위치 몬스터 ID 개수 좌표 리스폰 작업
맵을 선택하면 스폰 목록이 표시됩니다

➕ 커스텀 몬스터 추가

📖 항목 설명 보기 (클릭하여 펼치기)
  • 이름: 게임에 표시되는 몬스터 이름. 스폰 시 이 이름으로 지정됩니다.
  • name_id: 클라이언트용 이름/메시지 ID. $0 또는 $숫자 형태. 비우면 자동 생성.
  • gfx: 몬스터 외형 ID. 0=기본. 클라이언트 npc/몬스터 gfx와 맞추세요.
  • level: 레벨. 전투·경험치 계산에 사용.
  • hp / mp: 최대 체력·마나. 서버 밸런스 배율이 적용될 수 있음.
  • exp: 처치 시 주는 경험치.
  • str, dex, ac, mr, 보스 여부, 속성 저항 등은 DB에서 추가 수정 가능합니다.
게임 내에 표시되는 몬스터 이름. monster 테이블 name 컬럼이며, 스폰 시 이 이름으로 등록됩니다.
클라이언트에서 쓰는 이름/메시지 ID. 보통 $0 또는 $숫자. 비워두면 이름 기준으로 자동 생성됩니다.
몬스터 그래픽(외형) ID. 0이면 기본 외형. 클라이언트 npc/몬스터 gfx 목록과 맞추면 됩니다.
몬스터 레벨. 전투·명중·경험치 계산 등에 사용됩니다.
최대 체력(HP). 실제 적용 시 서버 밸런스(monster_hp_rate)가 곱해질 수 있습니다.
최대 마나(MP). 스킬 사용 등에 사용됩니다. 서버 밸런스(monster_mp_rate) 적용 가능.
플레이어가 이 몬스터를 처치했을 때 얻는 경험치입니다.

※ 힘(str)·민첩(dex)·방어(ac)·마방(mr)·보스 여부·크기(size)·속성 저항 등은 DB monster 테이블에서 추가로 수정할 수 있습니다.
